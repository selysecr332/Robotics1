# HW5 Path Planning — A* vs RRT/RRT*

## Обзор
В проекте реализован полный конвейер 2D-планирования пути в occupancy grid и выполнено сравнение трех алгоритмов:
- `A*` (сеточный поиск, 8-связность)
- `RRT`
- `RRT*` (с перестройкой дерева)

После нахождения маршрута применяется постобработка (сглаживание). В ноутбуке есть визуализация, метрики, анимации процесса поиска и аналитические выводы.

Главный файл:
- `HW5_Path_Planning.ipynb`

## Что реализовано
Сделано:
- Базовое сравнение: `A*` и `RRT`
- Улучшенное планирование: `RRT*`
- Сглаживание траекторий (gradient + collision-aware shortcut)
- Сравнение по метрикам:
  - длина пути
  - время вычисления
  - число посещенных узлов / точек
- Визуализация в Matplotlib:
  - карта
  - «сырой» и сглаженный пути
  - анимация роста дерева RRT
  - анимация расширения A* (explored cells)
- Исследование влияния разрешения карты на скорость A*
- Финальные выводы в конце ноутбука

## Источники карт
В ноутбуке можно переключать источник через `MAP_SOURCE`:
- `png` — собственные Ч/Б карты (`hw5_occ_map_01.png`, `hw5_occ_map_02.png`, `hw5_occ_map_03.png`)
- `movingai` — формат MovingAI `.map` (`hw5_movingai_sample.map`, размер `56x56`)
- `random_pr` — генерация случайных прямоугольных препятствий в стиле PythonRobotics

Единая конвенция occupancy grid:
- `grid[y, x] = 0` — свободно
- `grid[y, x] = 1` — препятствие

## Технические детали
### Загрузка карт
- PNG читается через OpenCV (`cv2.imread`) с fallback на PIL.
- Добавлен парсер MovingAI `.map`.
- Реализована инфляция препятствий (`cv2.dilate`).

### A*
- 8-связный поиск с диагональными переходами.
- Возвращает путь и метрики.
- Отдельная функция сохраняет кадры расширения для GIF.

### RRT
- Итеративный рост дерева со случайным сэмплированием.
- Goal bias и проверка коллизий для отрезков.
- Сохранение кадров ребер для анимации.

### RRT*
- Поиск соседей, выбор лучшего родителя, rewiring.
- Защита от циклов при восстановлении пути.

### Сглаживание
- Градиентное сглаживание узлов пути.
- Shortcut-сглаживание с проверкой проходимости.

## Краткие выводы
- Уменьшение разрешения карты ускоряет A* (меньше узлов), но может ухудшать качество пути и «закрывать» узкие проходы.
- RRT испытывает трудности в узких коридорах из-за малой вероятности удачных сэмплов.
- RRT* улучшает качество пути через rewiring, но не устраняет проблему исследования узких мест.
- Сглаживание снижает ломаность траекторий и часто уменьшает длину маршрута.
